immuneSIM: Comprehensive Details from ImmunRef Paper Analysis

1. Tool/Model Name:
immuneSIM

2. Reference(s):
- Weber CR, Akbar R, Yermanos A, et al. (2020) immuneSIM: tunable multi-feature simulation of B- and T-cell receptor repertoires for immunoinformatics benchmarking. Bioinformatics 36, 3594â€“3596
- R package version: 0.8.7 (as used in ImmunRef paper)

3. Category:
Immune Repertoire Simulation Tool, Benchmarking Framework, Ground Truth Generation

4. Main Purpose/Function:
immuneSIM is a tunable multi-feature simulation framework for generating B- and T-cell receptor repertoires that closely mimic real immune repertoire data. It is designed for immunoinformatics benchmarking and provides ground truth datasets for validating immune repertoire analysis tools.

5. Key Features:
- **Multi-species support**: Human and mouse repertoires
- **Multi-receptor support**: BCR (IgH) and TCR (TRB) simulation
- **Tunable parameters**: 8+ parameters for controlling repertoire characteristics
- **Native-like output**: Demonstrated to produce repertoires similar to real data
- **Post-simulation modifications**: Allows additional modifications to test specific scenarios
- **R package implementation**: Easy integration with analysis workflows

6. Input Requirements:
- Parameter configuration for 8 main parameters
- Germline gene sets (V, D, J genes)
- Species specification (human/mouse)
- Receptor type specification (BCR/TCR)
- Optional: Post-simulation modification parameters

7. Output:
- **Simulated repertoires**: 120,000 sequences per repertoire (as used in ImmunRef)
- **Native-like sequences**: CDR3 sequences with realistic properties
- **Ground truth data**: Known parameter values for validation
- **Multiple datasets**: 200+ repertoires with controlled parameter variations

8. Validation/Evaluation:
- **Native-likeness demonstrated**: Weber et al., 2019 shows similarity to real data
- **Used for benchmarking**: Validates immune repertoire analysis tools
- **Parameter sensitivity**: Tests tool sensitivity to controlled parameter changes
- **Feature-specific validation**: Each parameter affects specific repertoire features

9. Strengths:

Detailed Parameter Analysis (from ImmunRef paper):

A. Eight Main Parameters:
1. **Species**: Human vs. Mouse
2. **Receptor Type**: IgH (BCR) vs. TRB (TCR)
3. **Germline Gene Distribution**: V-, D-, J-gene frequency patterns
4. **Clone Count Distribution**: Controls clonal expansion patterns
5. **V-gene Frequency Noise**: Variation in V-gene usage
6. **D-gene Frequency Noise**: Variation in D-gene usage (BCR only)
7. **J-gene Frequency Noise**: Variation in J-gene usage
8. **Insertion/Deletion Likelihoods**: N-region and exonuclease trimming

B. Post-Simulation Modifications:
1. **Motif Implantation**: 
   - Motifs "YAY" ("tacgcctac") and "YVY" ("tacgtctac")
   - Implanted with 2.5% probability each
   - Random position in CDR3 region
   - Simulates antigen-binding motifs

2. **Hub Removal**:
   - Removes top 5% sequences with highest hub scores
   - Simulates variation in sequence similarity network architecture
   - Tests network-based analysis methods

3. **Synonymous Codon Changes**:
   - Replaces codons with synonymous alternatives
   - "tat"/"tac" for Tyrosine, "agt"/"agc" for Serine, "gtt"/"gtg" for Valine
   - Changes nucleotide composition while preserving amino acid sequence
   - Tests k-mer based analysis methods

C. Parameter Impact on immuneREF Features:
- **Clone count distribution**: Primarily affects diversity feature
- **Germline gene noise**: Affects VDJ usage feature
- **Insertion/deletion**: Affects multiple features including architecture
- **Species/receptor type**: Affects all features to varying degrees
- **Post-simulation modifications**: Test specific feature sensitivities

D. Simulation Scale (from ImmunRef):
- **200 simulated repertoires** with controlled parameter variations
- **120,000 sequences** per repertoire initially
- **Subsampled to 100,000** sequences for analysis
- **40 parameter combinations** over multiple replicates
- **Ground truth available** for all parameter values

E. Integration with immuneREF:
- **Baseline validation**: Default parameters produce >0.96 similarity for most features
- **Sensitivity testing**: Detects controlled parameter variations
- **Feature-specific validation**: Each parameter affects specific immuneREF features
- **Reference datasets**: Provides biological interpretation framework

F. Key Innovation:
- **Multi-feature simulation**: Captures multiple aspects of repertoire biology
- **Tunable parameters**: Allows systematic testing of analysis tools
- **Native-like output**: Realistic repertoires for meaningful validation
- **Post-simulation flexibility**: Enables testing of specific biological scenarios
